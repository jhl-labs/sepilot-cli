# Security Policy

## Supported Versions

| 버전 | 지원 상태 |
|------|----------|
| 0.1.x | :white_check_mark: 지원 |

## Reporting a Vulnerability

보안 취약점을 발견하셨다면 **공개 이슈로 등록하지 마시고**, 아래 방법으로 비공개 보고해주세요.

### 보고 방법

1. GitHub의 [Security Advisories](../../security/advisories/new)를 통해 비공개 보고
2. 또는 프로젝트 메인테이너에게 직접 연락

### 보고 시 포함할 내용

- 취약점에 대한 상세 설명
- 재현 단계
- 영향을 받는 버전
- 가능하다면 수정 제안

### 대응 절차

1. **확인** — 보고 접수 후 가능한 빠르게 확인 회신
2. **조사** — 취약점의 범위와 영향도 분석
3. **수정** — 패치 개발 및 테스트
4. **공개** — 수정 버전 릴리스 후 취약점 공개

### 주의사항

- API 키, 토큰 등 민감한 정보는 `.env` 파일에 보관하고, 절대 커밋하지 마세요.
- SEPilot은 Bash 명령을 실행할 수 있으므로, 신뢰할 수 없는 프롬프트 입력에 주의하세요.
